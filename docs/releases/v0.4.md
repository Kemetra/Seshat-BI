# Seshat BI -- v0.4 release line

This document records the repository candidates in the v0.4 release line. It
summarizes merged work honestly; it does not by itself prove PyPI, GitHub
Release, Claude catalog, Codex catalog, or OpenAI public-plugin availability.
Each publication path requires its own named-owner approval and external
evidence.

## Product identity

Seshat BI remains an **agent-first BI readiness system**. It requires evidence
and named-human approvals at readiness gates; it does not turn an agent
recommendation, a version number, or a release-candidate audit into an
approval.

## v0.4.1 patch candidate -- changes since v0.4.0

- **Activated the governed `dagster-dbt` engine seam** (spec 135, PR #307):
  `silver_tables` and `gold_tables` can select a dbt-backed build through a
  committed `mappings/<table>/build-engine.yaml` flag. The path remains behind
  the existing Seshat gates and runs through the governed `seshat.dbt` control
  layer into isolated shadow schemas. The unused `dagster-dbt` library pin was
  removed because no released version accepts the repository's `dbt-core`
  1.12 boundary and the execution path does not import it.
  **Honest label**: the dbt engine is a governed rehearsal with
  `warehouse_updated: false`; migrations remain the default, parity oracle,
  and rollback path. Live dbt drive remains `[PENDING LIVE PROFILE]`.
- **Governed dependency co-resolution and freshness gate** (spec 136, PR #308):
  `dependency-environments.yaml`, `scripts/dep_coresolve.py`, and the
  `dep-integrity` workflow now resolve every declared install environment and
  governed cross-product in an ephemeral environment. This catches dependency
  conflicts at merge time and adds a weekly advisory freshness report without
  changing pins or opening pull requests automatically. The previously
  unwatched `orchestration/dagster` pip environment is now covered by
  Dependabot.
- **Repository maintenance and funding updates** (PRs #309-#311 and #313):
  GitHub Sponsors is enabled and promoted in the README, while the Dagster
  smoke workflow moves to the current pinned `actions/checkout` and
  `actions/setup-python` releases.
- **Post-merge documentation and Dependabot reconciliation** (PR #314): stale
  pre-merge dependency-gate wording was corrected, the removed `dagster-dbt`
  dependency was removed from coverage comments, and Dependabot entries now use
  `labels: []` so absent labels are not recreated implicitly.

## v0.4.0 baseline

- **Governed dbt transformation MVP** (spec 133, PR #299): an exact optional
  `dbt-core==1.12.0` + `dbt-postgres==1.10.2` runtime (`seshat-bi[dbt]`), a
  tracked eight-model `retail_store_sales` shadow graph with 24 selected tests,
  Mapping Ready and source-map citation validation, immutable accepted plans,
  invocation locks, redacted subprocess handling, normalized parity evidence,
  the `seshat dbt` doctor/validate/plan/build/test/inspect-run command family,
  and the shared `dbt-workflows` Claude/Codex skill.
- **Dagster orchestration MVP** (spec 134 / F030, PR #300): a real
  `orchestration/dagster/` runtime project running the full 11-asset medallion
  graph behind fail-closed gates, read-only HUMAN-SEAM approval reads, and a
  publish wall that triggers the publish feature while failing closed when it
  is absent. It includes the `seshat dagster doctor|run|evidence` command family,
  derived schema-validated run evidence, and a definitions-load CI smoke test.
- **Canonical public command surface and normalized command names** (PR #297):
  `distribution/public-command-surface.yaml` is the authority for generated
  agent-bundle commands, backed by contract drift gates.
- **Eight Power BI plugin commands and the shared `powerbi-workflows` skill**
  (PR #297), generated for both the Claude and Codex bundles.
- **`dagster-workflows` public skill and Dagster surface parity** (PR #303): the
  governed Dagster workflow ships as a dedicated shared skill in both bundles,
  routed from the three Dagster commands and the `seshat-bi` router.
- **Bundle provenance hardened against squash-merges** (PRs #301 and #302):
  everyday regeneration tolerates rewritten feature history while the
  coordinated-release audit retains strict ancestry requirements.

## What this release line does NOT claim

- No live dbt compile/build/test/parity evidence exists yet; live-path claims
  remain `[PENDING LIVE PROFILE]`.
- Dagster schedules and sensors remain STOPPED; enabling unattended execution
  is a named-human action.
- The activated dbt engine seam writes only to isolated shadow schemas and does
  not replace the governed migration path.
- No publication action (PyPI, GitHub Release, Claude/OpenAI catalogs) is
  performed or implied by this document.
